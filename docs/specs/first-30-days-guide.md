# 첫 30일 가이드 콘텐츠 스펙

> **위상**: v8 §4 사용자 여정 4·5단계 (*시작* + *유지*) 의 구체화. 페르소나 B가 *회원가입 직후 30일 동안* 어떤 행동을 *어느 시점에* 하면 좋은지 안내.
>
> **핵심 가정**: 페르소나 B는 *시간 부족 + 입문자가 아닌 척하는 입문자*. 한 번에 모든 걸 시키면 이탈. **주 1회 한 가지 행동**으로 압축.
>
> **마일스톤**: M6 (베타).

---

## 1. 핵심 원칙

1. **주 1회 한 가지** — 한 주에 하나만. 두 개 이상 동시 X.
2. **각 단계 5~15분 이내** — 페르소나 B의 *주말 자투리 시간*에 맞춤.
3. **누적 만들지 않기** — 놓친 주는 *안 한 채로 OK*. 체크리스트 강박 X.
4. **격려 톤** — 매 단계 끝에 *"잘했어요"* 작은 축하.
5. **계좌 없으면 인라인 가이드** — 별도 화면 분기 X. 체크박스 옆에서 *"이 계좌 아직 없어요"* 옵션.
6. **회사 DC는 별도 트랙** — 회사 운용 플랫폼에서 매매하는 흐름이 다름.

---

## 2. 4주 흐름 (Week 1 ~ Week 4)

### Week 1 — *지금 가진 것 정리하기* (10분)

**목표**: 현재 자산 현황을 *도토리에 한 번만* 입력.

**행동**:
- 사용 중인 계좌 등록 (ISA·연금저축·IRP·DC·일반 위탁·CMA)
  - 계좌 없으면 인라인 가이드 → 다음 주에 만들기 (행동 미루기 OK)
- 보유 종목·수량 입력 (또는 스크린샷 OCR — 베타에서는 수기)
- DC 계좌의 *현재 운용 라인업* 확인 (회사 운용 플랫폼 캡처)

**카피 (인앱)**:
> *"첫 주에 할 일은 하나예요. 지금 가지고 있는 계좌·종목을 도토리에 한 번 입력하기. 이미 ISA·연금저축이 있다면 5분, 처음이라면 다음 주로 미뤄도 괜찮아요."*

**완료 시 카피**:
> *"잘했어요. 도토리가 이제 당신의 자산 풍경을 봐요. 다음 주에는 *없는 계좌*를 만들어볼게요. (이미 다 있다면 Week 2는 건너뛰기)"*

---

### Week 2 — *없는 계좌 만들기* (15분)

**목표**: 추천 포트폴리오 실행에 *필요한 계좌* 중 없는 것을 만든다.

**우선순위 가이드** (페르소나 B 기준):
1. **ISA** — 비과세 한도 200만원. 가장 진입 쉬움. 증권사 모바일 앱에서 비대면 5분.
2. **연금저축** — 세액공제 효과 최대. 30대에 가입 시 복리 효과 큼.
3. **IRP** — 회사 IRP가 이미 있다면 *추가 납입* 활용. 없다면 별도 가입.
4. **DC 운용 시작** — 디폴트 옵션이라면 *원리금 보장형 → 자산배분형*으로 변경.

**카피**:
> *"노후 자산배분의 기본은 *세금 혜택을 받는 계좌*에 먼저 담는 거예요. 이번 주는 ISA부터 만들어볼까요? 비대면으로 5분이면 끝나요."*

**인라인 가이드** (각 계좌별):
- 추천 증권사 3곳 비교 (수수료·이벤트·UX) — 단 *"이 증권사를 가입하세요"* 카피 금지 (자본시장법)
- 비대면 가입 절차 캡처 가이드
- 가입 후 도토리 인앱에 *"이 계좌 만들었어요"* 체크 → 자동 등록 안내

**완료 시 카피**:
> *"한 알 모았네요. 다음 주에는 첫 매매 지시서를 받아볼게요."*

---

### Week 3 — *첫 매매* (20분)

**목표**: 위험 성향 진단 결과 추천된 포트폴리오로 *첫 매매*.

**행동**:
1. 위험 성향 진단 결과 다시 확인 (회원가입 시 진행, 변동이 있으면 재진단)
2. 추천 포트폴리오의 *카테고리별 종목* 선택 (3~5개 후보 중)
3. 매매 지시서 생성: *"어느 계좌에서 어느 종목 몇 주씩 사세요"*
4. 증권사 앱(또는 회사 운용 플랫폼)에서 그대로 매매
5. 도토리 *"매매 완료"* 버튼 → 보유 수량 일괄 반영

**카피**:
> *"이번 주는 첫 매매. 도토리가 *어느 계좌에 어느 종목 몇 주*까지 정리해드려요. 증권사 앱에서 그대로 따라 매매하시면 돼요. 5종목 매매에 약 15분."*

**인라인 도움말**:
- 종목 선택 시 *TER, 운용사, 추적 오차* 인라인 팝오버 (용어 사전 연결)
- 한 번에 한 계좌씩 매매 안내 (정신적 부하 분산)

**완료 시 카피**:
> *"한 알이 채워졌어요. 도토리는 이제부터 *덜 보이는 모습*으로 바뀔 거예요. 평소엔 조용히, 필요할 때만 알려드릴게요."*

---

### Week 4 — *매월 자동화 셋업* (10분)

**목표**: 매월 적립 + 분기 리밸런싱 *자동화*.

**행동**:
1. 매월 적립 금액 설정 (월급일 기준 자동이체 안내)
2. 적립 매수 대상 비중 확인 (드리프트 자동 보정)
3. 알림 설정:
   - 월급일 적립 리마인더 ON (사용자 선택)
   - 드리프트 ±5%p 초과 시 주간 요약 메일 ON
   - 그 외 알림 OFF (*"덜 보게 만드는 앱"* 철학)

**카피**:
> *"4주 차예요. 이제 도토리가 자동으로 굴러가요. 매월 월급일에 적립 매수만 하시면 되고, 비중이 너무 흔들릴 때만 도토리가 살짝 알려드릴게요. 평소에는 도토리도, 당신도 한숨 돌리세요."*

**완료 시 카피**:
> *"30일 동안 정말 잘했어요. 한 알, 두 알, 세 알, 네 알 — 도토리 보관소가 단단해지고 있어요. 다음 분기 리포트를 기다려주세요."*

---

## 3. 추가 콘텐츠 (옵션 — Week 5 이후)

페르소나 B가 *4주를 다 완료한 후* 노출되는 *학습형 콘텐츠*. 강요 없이 *대시보드 카드*로만.

| 콘텐츠 | 분량 | 결 |
|---|---|---|
| *"왜 자산배분이 부동산·개별주보다 안전한가"* | 600자 | 본질·격려 |
| *"인플레이션이 노후에 미치는 영향"* | 500자 | 사실 + 작은 우려 |
| *"리밸런싱은 왜 분기 1회면 충분한가"* | 500자 | 안심 |
| *"세액공제 한도 활용 가이드"* | 800자 | 실용 |
| *"ISA 만기 시나리오 비교"* | 800자 | 의사결정 도구 |
| *"DC 운용을 어떻게 바꿔야 할까"* | 1,000자 | 페르소나 B의 핵심 통점 |

## 5. 이탈 핸들링

페르소나 B의 이탈은 *놓친 주를 사용자에게 압박하지 않는 것*이 핵심.

| 시나리오 | 대응 |
|---|---|
| Week 1 미완료, 7일 경과 | 가벼운 리마인더 1회 (*"천천히 하셔도 괜찮아요"*) |
| Week 2 미완료, 7일 경과 | 가벼운 리마인더 1회. 이후 *침묵*. |
| Week 3 미완료, 14일 경과 | 짧은 격려 메일 1회 (*"한 알이라도 시작이에요"*). 이후 *침묵*. |
| Week 4 미완료 | 알림 X — *4주 가이드는 끝남*. 사용자가 돌아오면 자동 재개. |
| 30일 후 가이드 미완료 | 가이드 카드를 *"천천히 보기"* 모드로 유지. 강요 X. |

> **금기**: *"X일째 미루고 있어요"*, *"늦으면 손해예요"* 같은 압박 카피.

## 6. 데이터 모델 (구현 시)

```typescript
// src/data/first-30-days.ts (제안)

export interface GuideStep {
  week: 1 | 2 | 3 | 4;
  title: string;
  goal: string;             // 한 줄
  estimatedMinutes: number; // 5, 10, 15, 20
  actions: string[];        // 체크박스 목록
  intro: string;            // 카피 (인앱)
  completionCopy: string;   // 완료 시 카피
  inlineGuides?: string[];  // 인라인 가이드 key
}

export interface GuideProgress {
  userId: string;
  startedAt: Date;
  completedSteps: number[]; // [1, 2, 3, 4]
  skippedSteps: number[];   // 사용자가 *"건너뛸게요"* 선택한 단계
  // 미완료는 명시적으로 추적 X (압박 회피)
}
```

## 7. 다람쥐·도토리 메타포 적용

이 가이드는 *감성 화면*에 가깝다 (v8 §3.4). 메타포 자연스럽게 사용:

- *"한 알 모았네요"* (Week 2 완료)
- *"한 알이 채워졌어요"* (Week 3 완료)
- *"도토리 보관소가 단단해지고 있어요"* (Week 4 완료)
- 마스코트 일러스트는 각 주 완료 카드에 *작게* 노출

다만 운영 화면(매매 지시서·세금 한도)은 메타포 X. 정확한 용어.

| 이메일 vs 푸시 | 둘 다 / 이메일만 / 푸시만 | **둘 다 + 사용자 선택권** (디폴트 둘 다 OFF, 가입 시 ON 안내) |
