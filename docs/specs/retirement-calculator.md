# 은퇴 계산기 스펙 (Retirement Calculator)

> **위상**: v8 §4 사용자 여정 1단계 *"wake-up"*의 진입 훅. 비로그인 SEO 미끼 페이지(`/tools/retirement`)이자 회원가입 깔때기의 시작점.
>
> **페르소나 B 기준**: 직장인 3~5년차, 시간 부족, 막연한 노후 불안, *"입문자가 아닌 척하는 입문자"*. 입력 부담 최소화 + 격려형 결과 + 부드러운 회원가입 CTA.
>
> **마일스톤**: M6 (경량 v1) → M9–M10 (정밀화).

---

## 1. 핵심 원칙

1. **5개 이하 입력** — 페르소나 B는 *7개 이상이면 이탈*. 기본 5개 + 고급은 접어둠.
2. **격려형 카피** — *"X억 부족!"*은 안 씀. *"이미 한 발 시작했어요. 나머지가 이거예요."* 결.
3. **법적 안전지대** — 일반 가정 기반 *시뮬레이션*임을 명시. 종목 추천 절대 X (v7 §4.3).
4. **비로그인 가능** — 결과 본 후 *부드러운* 회원가입 유도. SEO 미끼.
5. **모바일 퍼스트** — 페르소나 B는 출퇴근 중 휴대폰으로 만남.
6. **즉시 반응** — 입력 변경 시 결과 실시간 업데이트 (서버 호출 0).

## 4. 계산 로직

### 4.1 시스템 가정 (디폴트)

- **실질 수익률 (r)**: 연 4% (자산배분 평균, 인플레 차감 후)
- **인플레이션 (i)**: 연 2.5% (참고용 — 실질 수익률을 직접 사용하므로 이중 차감 X)
- **인출률**: 4% (Trinity 룰 보수 변형)
- **인출 기간**: 25년 (65→90세)

### 4.2 계산식

**(A) 은퇴 시점 필요 자산 (실질 가치)**

```
필요 자산 = (월 생활비 × 12) ÷ 인출률
예: (250만원 × 12) ÷ 0.04 = 7.5억
```

**(B) 현재 페이스로 도달하는 자산 (실질 가치)**

```
n = 은퇴 희망 연령 − 현재 나이 (년)
r_월 = (1 + 0.04)^(1/12) − 1 ≈ 0.327%

도달 = 현재자산 × (1+r)^n + 월저축 × 12 × [((1+r)^n − 1) ÷ r]
```

**(C) 부족분 / 잉여분**

```
부족 = max(0, A − B)
잉여 = max(0, B − A)
```

**(D) 따라잡기 시나리오 — 월 추가 저축액**

```
부족이 양수일 때:
월_추가 = 부족 ÷ (12 × [((1+r)^n − 1) ÷ r])
```

**(E) 수익률 개선 시나리오 — r' = 5% (자산배분 도입 효과)**

```
(B'): r 대신 0.05로 (B) 재계산
효과 = B' − B → "자산배분으로 평균 +1%p 가정 시 N억 추가 도달"
```

### 4.3 엣지 케이스

| 케이스 | 결과 카피 |
|---|---|
| 잉여분 발생 | *"이미 충분해요. 65세에 약 N억까지 가능해서 더 일찍 은퇴도 가능해요."* |
| 현재 자산 0 | *"걱정 마세요. 0에서 시작해도 늦지 않아요. 매달 W만원이면 따라갈 수 있어요."* |
| 현재 나이 ≥ 은퇴 연령 | 입력 오류 인라인 안내 |
| 부족이 너무 커서 월 추가 저축이 비현실적 (예: 월급 초과) | *"은퇴 시점을 N년 늦추면 따라잡을 수 있어요"* 시나리오 노출 |
| 입력 누락 | 슬라이더·숫자에 기본값 항상 박혀 있어 *결과는 항상 산출됨* |

### 5.2 카피 결

**쓰는 결**:
- *"N억이 필요해요"* (사실 진술, 숫자 그대로)
- *"이미 한 발 시작했어요"* (격려)
- *"매달 W만원만 더 모으면 따라잡을 수 있어요"* (구체적 다음 행동)
- *"걱정 마세요. 늦지 않았어요."*
- *"두 가지 길이 있어요"*

**안 쓰는 결**:
- ❌ *"부족!"* (느낌표·경고 어조)
- ❌ *"이대로면 노후 파산"*
- ❌ *"지금 시작하지 않으면 늦습니다"*
- ❌ *"위험"* / *"불안"* / *"문제"*

### 5.3 시나리오 카드 (3개 → 2개로 축약 권장)

원안: 현재 페이스 / +월 추가 저축 / +자산배분
**최종 권장**: *현재 페이스* + *도토리와 함께* (자산배분 + 월 추가 저축을 묶음)

→ 페르소나 B는 *비교·선택*보다 *명확한 한 길*을 더 선호. 단순화.

### 5.4 다람쥐 일러스트 가이드

- 결과 화면 *상단 좌측 또는 우측* 작게 (메인이 아님)
- 입력 화면은 일러스트 X (운영 화면에 가까움)
- 가을 숲 결 베이지·어스 톤 (디자인 시스템 토큰 보강 후 적용)

## 8. UX 디테일

### 8.1 입력 인터랙션

- 슬라이더 + 숫자 입력 *동시 노출* (둘 중 어느 쪽으로도 조작 가능)
- 입력 변경 시 *실시간 결과 업데이트* (디바운스 200ms)
- 진행률 인디케이터: *"5개 중 3개"*
- *"입력은 5개로 충분해요"* 안심 카피 (입력 시작 화면 상단)

### 8.2 자동 저장 / 복귀

- 입력 중 localStorage 저장 (key: `dotori-retirement-input-v1`)
- 페이지 이탈 후 복귀 시 *"이어서 하기"* 토스트
- 회원가입 후 → 서버 DB로 마이그레이션 (사용자 프로필에 영구 저장)

### 8.3 공유 (개인정보 주의)

- query string에 평문 자산액 인코딩 ❌
- 대신: *대략 구간*만 인코딩 (`assets_band=1to3`, `1억미만`/`1-3억`/`3-5억`/`5억이상` 등 4구간)
- 또는: 공유 시 *결과 자체*는 일반화하고 *"이 계산기로 직접 해보기"* 링크만

### 8.4 인쇄 / 저장

- 결과 화면 print-friendly CSS
- *"이미지로 저장"* 버튼 (간단한 OG 카드 형태) — 인스타·카톡 공유용

### 8.5 접근성

- 슬라이더 키보드 조작 가능
- 스크린 리더용 aria-label
- 색상 대비 WCAG AA 이상

## 10. 기술 스택

- **렌더링**: Next.js App Router. 서버 컴포넌트 + 클라이언트 컴포넌트 (입력은 클라이언트).
- **계산**: 클라이언트 사이드 순수 함수 (`src/lib/retirement-calc.ts`). 서버 호출 0.
- **저장**: 비로그인 = localStorage. 로그인 = `users.retirement_input` (또는 별도 테이블).
- **시각화**: 기존 차트 컴포넌트 재사용 (`*.chart.tsx`). DS 컬러 우선, hex 리터럴 허용 (eslint exempt).
- **테스트**: `src/__tests__/lib/retirement-calc.test.ts` — 정상·경계·엣지 (잉여·0 자산·이미 은퇴) 커버.
