# 인라인 용어 사전 스펙

> **위상**: v8 §1.3 페르소나 B (*"입문자가 아닌 척하는 입문자"*)에게 *입문자 취급 없이* 용어를 풀어주는 인라인 도움말.
>
> **UX**: 본문 내 용어에 *점선 밑줄* + 호버/탭 → 작은 팝오버. 별도 사전 페이지로 강요 X.
>
> **데이터 위치 (구현 시)**: `src/data/glossary.ts`.

---

## 1. 원칙

1. **30개 핵심 용어**로 시작. 매년 1~2회 보강.
2. **2~3문장 이내** — 페르소나 B는 *긴 설명에 이탈*.
3. **친구 다람쥐 톤** — *"이건 ___이에요"* (선생님 톤 X).
4. **예시 한 줄** — 추상 정의보다 *"예: ___"* 가 더 빨리 이해됨.
5. **연관 용어 링크** — 호버 안에서 다음 용어로 점프 가능.

---

## 2. 카피 톤 가이드

**OK 결**:
- *"___이에요"* (담백)
- *"쉽게 말하면 ___"*
- *"예: ___"*

**안 쓰는 결**:
- ❌ *"___을 의미합니다"* (사전식, 거리감)
- ❌ *"초보자도 알기 쉽게"* (입문자 취급)
- ❌ *"반드시 알아야 할 ___"* (강요·불안 자극)

## 3. 핵심 용어 30선

### 3.1 계좌·세금 (8개)

| 용어 | 설명 | 예시 / 비고 |
|---|---|---|
| **ISA** (개인종합자산관리계좌) | 한 계좌 안에서 주식·ETF·예금 등을 모아 굴리고, 수익 200만원까지 비과세 받는 절세계좌. | 연 2,000만원·총 1억까지 납입 가능. |
| **연금저축** (연금저축펀드/보험) | 노후 대비 장기 저축계좌. 매년 납입액 일부에 *세액공제*가 들어와요. | 연 600만원까지 세액공제. 만 55세 이후 인출. |
| **IRP** (개인형 퇴직연금) | 직장에서 받은 퇴직금이나 본인 추가 납입금을 굴리는 계좌. 연금저축과 합쳐 *세액공제 한도 900만원*. | 회사가 만들어준 IRP에 본인이 추가 납입 가능. |
| **DC** (확정기여형 퇴직연금) | 회사가 매월 적립금을 넣어주고, *직접 운용*해야 하는 퇴직연금. 디폴트 옵션에 두면 거의 안 굴러가요. | 약 90%가 디폴트 옵션 방치. |
| **DB** (확정급여형 퇴직연금) | 회사가 직접 운용하고 *정해진 금액*을 받는 퇴직연금. DC와 반대. | 운용 책임이 회사에 있음. |
| **세액공제** | 낸 세금 중 일부를 돌려받는 것. 연금저축·IRP에 넣으면 연 최대 약 100만원 환급. | 13월의 월급. |
| **과세이연** | 지금 안 내고 나중에 낼 수 있게 미루는 것. 연금계좌 안에서는 매매차익에 세금이 안 붙어요. | 복리 효과 ↑ |
| **분리과세** | 다른 소득과 합치지 않고 *따로* 낮은 세율로 내는 것. 연금소득은 1,500만원까지 분리과세. | 종합소득세 합산보다 유리. |

### 3.2 자산배분·투자 (7개)

| 용어 | 설명 | 예시 / 비고 |
|---|---|---|
| **자산배분** | 주식·채권·금 같은 *서로 다르게 움직이는* 자산을 비율로 섞는 것. 한쪽이 떨어져도 다른 쪽이 받쳐줘요. | 주식 60% + 채권 40% 등. |
| **ETF** | 여러 종목을 한 바구니로 묶어 *주식처럼 거래*하는 상품. 분산 투자가 한 번에. | KODEX 200 = 코스피 상위 200개를 한 번에. |
| **TDF** (Target Date Fund) | 은퇴 목표 연도에 맞춰 *시간이 갈수록 자동으로 안전해지는* 펀드. | 2055 TDF = 2055년 은퇴 가정. |
| **리밸런싱** | 시간이 지나 *비율이 틀어진 자산*을 다시 원래 비율로 맞추는 것. 분기·연 1회. | 60/40에서 65/35로 변하면 다시 60/40으로. |
| **드리프트** | 시장 변동 때문에 자산 비율이 *원래 목표에서 벗어난 정도*. ±5%p 넘으면 리밸런싱. | "주식이 +7%p 드리프트 중" |
| **분산 투자** | 여러 자산·지역·종목에 *나눠서* 투자하는 것. 한 곳이 망해도 전체가 망하지 않게. | 자산배분의 핵심 원리. |
| **매매 지시서** | 도토리가 *어떤 종목을 몇 주 사고 팔지* 정리해주는 메모. 사용자는 증권사 앱에서 그대로 따라 매매. | 도토리는 매매 자체는 하지 않음. |

### 3.3 수익률·성과 (5개)

| 용어 | 설명 | 예시 / 비고 |
|---|---|---|
| **TWR** (시간가중수익률) | 입출금 영향을 빼고 *순수한 자산 운용 성과*만 본 수익률. 펀드 비교에 적합. | "이 자산배분이 연 7% TWR" |
| **MWR** (금액가중수익률) | 입출금 시점·금액까지 반영한 *내가 실제로 번 돈* 기준 수익률. | "내 돈 기준으로는 연 5%" |
| **YTD** (Year To Date) | 올해 1월 1일부터 오늘까지의 수익률. | "YTD +8.2%" |
| **연환산 수익률** | 짧은 기간 수익률을 *연 단위로 환산*한 값. 비교 기준. | 6개월 5% → 연환산 약 10% |
| **샤프 지수** | 위험 대비 수익률 효율. *높을수록 잘 굴렸다*는 뜻. | 1.0 이상이면 양호. |

### 3.4 시장·운영 (5개)

| 용어 | 설명 | 예시 / 비고 |
|---|---|---|
| **TER** (운용보수) | ETF가 *매년 자동으로 떼어가는 비용*. 낮을수록 좋아요. | KODEX 200 TER = 0.15% (연) |
| **추적 오차** | ETF가 *원래 따라가려는 지수*에서 얼마나 벗어났는가. 작을수록 좋음. | "추적 오차 0.3%" |
| **시가총액** | 회사 가치 = 주가 × 발행 주식 수. ETF는 *순자산총액(NAV)* 으로 봄. | 시총 큰 ETF가 거래 안전. |
| **유동성** | 사고 팔기 *얼마나 쉬운가*. 거래량 적으면 호가 차이로 손해. | 일평균 거래액 1억 이상 권장. |
| **벤치마크** | 비교 기준이 되는 *대표 지수*. 코스피, S&P 500, 60/40 포트폴리오 등. | "내 수익률 vs 코스피" |

### 3.5 은퇴·노후 (5개)

| 용어 | 설명 | 예시 / 비고 |
|---|---|---|
| **인출률** | 은퇴 후 매년 자산의 *몇 %씩 꺼내 쓸지* 의 비율. 4%가 표준 가정. | 7.5억 × 4% = 연 3,000만원 |
| **4% 룰** | 은퇴 자산의 4%씩 인출하면 *25년 이상 안전*하다는 경험 법칙. | Trinity Study (1998) |
| **인플레이션** | 물가가 매년 오르는 정도. 미래 1억은 지금 1억보다 *적게* 살 수 있어요. | 한국 평균 약 연 2~2.5% |
| **실질 수익률** | 인플레이션을 *뺀* 진짜 자산 증가율. 명목 6% − 인플레 2% = 실질 4%. | 도토리 계산기는 실질 4% 가정. |
| **국민연금** | 국가에서 의무 가입해 *노후에 매월 받는* 연금. 은퇴 자산 계산에 함께 반영해야 정확. | 평균 월 60만원 (2024년) |

## 5. UX 디테일

### 5.1 인라인 노출

- 본문 내 용어 첫 등장 시: `<Term>ISA</Term>` 컴포넌트로 감싸 *점선 밑줄* + 호버/탭 → 팝오버.
- 같은 화면 내 *반복 등장은 첫 1회만* 표시 (시각 노이즈 방지).
- 운영 화면(매매 지시서·세금 한도 등)에서는 *밑줄 없음* — 이미 안다고 가정.

### 5.2 팝오버 디자인

```
┌─────────────────────────────────────┐
│  ## ISA (개인종합자산관리계좌)        │
│                                      │
│  한 계좌 안에서 주식·ETF·예금 등을    │
│  모아 굴리고, 수익 200만원까지        │
│  비과세 받는 절세계좌예요.            │
│                                      │
│  예: 연 2,000만원·총 1억까지 납입.     │
│                                      │
│  [관련: 연금저축, 세액공제]           │
└─────────────────────────────────────┘
```

### 5.3 검색

- 별도 사전 페이지 X (페르소나 B의 시간 자원 부족).
- 단, 헤더 검색바에서 용어 입력 시 *해당 본문 위치로 점프* 가능.

## 6. 운영

### 6.1 보강 시점

- M6 클로즈드 베타: 30개 코어
