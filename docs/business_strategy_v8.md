> **문서 위상**: v7과 *공존하며 재정렬*하는 기준서.
> v6 (`business_design_v6.md`) = 제품/데이터/기능 골격

## 1. 페르소나

### 1.1 1번 페르소나 — 직장인 3~5년차 (페르소나 B)

| 속성 | 값 |
|---|---|
| 나이 | 28~32세 |
| 상태 | 회사 IRP·DC 일부 보유, ISA·연금저축 안 만듦 |
| 시간 자원 | 자산 관리에 *매월 5분 이상 쓰기 어려움* |
| 자산 인식 | DC가 디폴트 옵션에 방치 중 (직장인 약 90%). 본인은 *그것조차 잘 모름* |
| 불안의 결 | *상대적 박탈감* > 절박함. *"다들 한다는데 나만 안 함"* |
| 미디어 피로 | 부동산·개별주·시장 추적 콘텐츠에 지침 |
| 결제 능력 | 월 5,000~10,000원 지출 가능 |
| 든든·일임 인식 | *수수료 비싸 보임* + *내가 못 하는 게 아닐 텐데* 사이의 망설임 |

### 1.2 보조 페르소나

- **A. 사회초년생 (23~28세)** — 절세계좌 거의 없음, *"어디서 시작?"*. B 다음 큰 세그먼트.
- **C. 30대 중반 (32~37세)** — 결혼·출산 직전, 일부 계좌 방치, *"관리할 시간이 없음"*.

> A·C는 *마케팅 깔때기로 흡수*하되, **제품 카피·UX 톤은 B를 기준**으로 잡는다.

### 1.3 페르소나 B의 결정적 특징 — *"입문자가 아닌 척하는 입문자"*

회사 DC를 통해 *펀드를 한 번은 골라본* 경험이 있지만 실제로는 디폴트에 방치. 너무 *"초보자용"*으로 노출하면 자존심 건드려서 이탈. 대신 **정상화 카피**가 답:

> *"퇴직연금 90%는 디폴트 옵션에 그대로 두고 있어요. 당신만 그런 게 아니에요."*

## 2. 동기 — 왜 만드는가

> *부동산도, 개별주도, 시장 추적도 못 하는 청년들이 — 그래서 그냥 미루고 방치하고 있는 게 안타까워서.*
>
> *자산배분은 평균을 따라가되 누구나 할 수 있는 가장 장벽 낮은 방법. 이걸 친구처럼 옆에서 알려주고 따라하게 만드는 도구가 비어 있어서 도토리를 만든다.*

이 동기는 *카피로 직접 노출되지는 않는다*. 카피는 페르소나 B에게 와닿는 결로 번역되어야 함 (§7).

## 3. 브랜드 음성 · 메타포

### 3.1 화자 위치

**옆자리 친구 다람쥐** — 선생님 아님, 전문가 아님, 은행원 아님, 재무설계사 아님.

### 3.2 음성 표

| 갈래 | 도토리 | 안 맞는 결 |
|---|---|---|
| 화자 위치 | 옆자리 친구 다람쥐 | 선생님·전문가·은행원 |
| 감정 자극 | 정상화·격려 (*"당신만 그런 게 아니에요"*) | 불안 자극 (*"X억 부족!"*) |
| 자존감 | 작은 승리 (*"한 발은 이미 시작했어요"*) | 입문자 취급 (*"기초부터 알려드릴게요"*) |
| 약속 | 조용한 자신감 (*평균이지만 단단한 길*) | 과장된 수익 약속 |
| 자주 쓰는 단어 | *함께 · 한 알씩 · 천천히 · 충분해요 · 늦지 않았어요* | *반드시 · 즉시 · 무조건 · 폭등 · 비결* |

### 3.3 다람쥐·도토리 메타포 — *외부/내부 분리 원칙*

| 레이어 | 메타포 | 톤 |
|---|---|---|
| **외부 메시지** (랜딩 H1·광고·SEO 미끼) | ❌ 사용 안 함 | 일반·설득력 있는 메시지 |
| **내부 경험** (온보딩·진단·축하·대시보드 인사·분기 리포트) | ✅ 자연스럽게 녹임 | 따뜻한 친구 다람쥐 |

> 외부 메시지는 *설득의 일*을, 내부 경험은 *온기·정체성의 일*을 한다. 메타포에 기대지 않은 카피가 외부에서는 더 강하다.

### 3.4 화면 결 분리 — *감성 vs 운영*

| 화면 결 | 메타포 | 톤 |
|---|---|---|
| **감성** (랜딩, 온보딩, 은퇴 계산기 결과, 첫 매매 축하, 분기 인사) | ✅ | 친구 다람쥐 |
| **운영** (매매 지시서, 세금 한도, 수익률, 계좌 상세, 결제) | ❌ | 차분한 도구 |

> 귀여움이 모든 화면에 깔리면 진지한 운영 화면에서 신뢰가 깎인다. 메타포는 *의식적으로* 감성 화면에만.

### 3.5 용어 변환 (감성 화면 한정)

| 정식 용어 | 감성 화면용 |
|---|---|
| 은퇴/노후 | *다음 계절* |
| 적립·매수 | *도토리 모으기* |
| 리밸런싱 | *도토리 정리* |
| 포트폴리오 | *도토리 바구니* |
| 계좌 | *도토리 보관소* (필요시) |

> 운영 화면에서는 정식 용어. 결제·세금·매매 지시서 화면에서 다람쥐 용어가 섞이면 신뢰 하락.

### 3.6 비주얼 톤 (한 줄)

따뜻한 베이지·어스 톤 + 약간의 가을 숲 그린, 둥근 모서리, 가벼운 폰트 두께. 마스코트 도토리·다람쥐 일러스트는 감성 화면에만, 과하지 않게.

> 디자인 시스템과의 정합성은 별도 검토 필요 (`docs/design-system.md`, `src/app/globals.css` 토큰).

## 4. 핵심 사용자 여정 (5단계)

| # | 단계 | 화면 | 핵심 경험 |
|---|---|---|---|
| 1 | **wake-up** | 은퇴 계산기 | *얼마 부족한지* 직시 |
| 2 | **납득** | 인라인 학습·짧은 가이드 | 자산배분이 *현실적 해법*인 이유 |
| 3 | **선택** | 위험 성향 진단 (2~3문항) | 1~2개 포트폴리오 추천 |
| 4 | **시작** | 보유 계좌 입력 → 첫 매매 지시서 | 첫 매매 (회사 플랫폼·증권사 MTS에서) |
| 5 | **유지** | 월급일·분기 알림 | 평소엔 침묵, 편차 있을 때만 |

### 단계별 설계 원칙

1. **wake-up — 직시, 다만 충격이 아닌 격려로**
   계산기 결과는 *"X억 부족합니다"* 보다 *"이미 한 발 시작했어요. 나머지가 이거예요."* 톤.
2. **납득 — 짧게**
   교양 콘텐츠는 *인라인·짧게·필요한 곳에만*. 별도 학습 페이지 강요 금지.
3. **선택 — 마비 방지**
   2~3문항 진단 후 *1~2개 추천*. 4개 다 비교 보여주는 모드는 *고급 옵션*.
4. **시작 — 마찰 인라인 해결**
   계좌 없으면 *인라인 가이드* (별도 화면 분기 X). 첫 매매 지시서는 가장 단순한 형태로.
5. **유지 — 침묵이 기본**
   편차 없으면 침묵. 월급일·분기 외엔 알림 0.

## 5. 제품 약속 4단

1. **진단** — 막연한 노후 불안 → *구체적 숫자* (은퇴 계산기)
2. **처방** — 검증된 자산배분 중 당신에게 맞는 1~2개 (3~4개 큐레이션 + 위험 성향 진단)
3. **실행** — *매달 5분*이면 끝나는 매매 가이드 (매매 지시서 패턴)
4. **유지** — 평소엔 조용히, 편차 있을 때만 (*"덜 보게 만드는 앱"*)

## 6. 포지셔닝

### 6.1 한 문장 비유

> *"든든이 기사 딸린 차라면, 도토리는 운전 학원 + 내비게이션이다. 면허 없는 청년이 첫 운전을 안전하게 시작하는 곳."*

### 6.2 시장 4축

| 경쟁 | 그들 | 도토리 |
|---|---|---|
| **든든** | 기사 딸린 차 (수수료 0.3~1%) | 운전 학원 + 내비 (월 4,900~9,900) |
| **토스·뱅샐** | 내비만 있는 차 (어디로 갈지 모름) | 학원 + 내비 + 첫 30일 동승 |
| **스노우볼72** | 시뮬레이터 (이론 완벽) | 실제 도로 |
| **더리치** | 시세 트래커 | 자산배분·세금 축 |

### 6.3 든든과의 차별 — *구조적 해자*

든든은 교육·자율성·다양한 포트폴리오를 *기술적으로 못해서* 안 주는 게 아니라 **그들의 BM이 그러면 망해서**다. 일임 수수료 모델은 사용자의 자율성·이해도가 올라가는 순간 이탈로 직결.

→ *"충분히 가르치는 자산배분 도구"* 는 든든이 구조적으로 절대 못 따라오는 영역. 이게 도토리의 진짜 해자.

## 7. 카피 프레임 (잠정 — 감성 다듬기 필요)

> ⚠️ 결은 잡혔으나 *2% 감성*이 부족. 이후 카피 다듬기 작업 필요.

### 7.1 핵심 결 — *정상화·격려*

페르소나 B의 *상대적 박탈감 + 막막함 + 미디어 피로 + 시간 없음*을 한 결로 묶는 답: *"막연 → 분명"*.

### 7.2 슬로건 후보 (랜딩 메인)

- *"막연한 노후, 분명한 한 걸음."* (잠정)

### 7.3 한 줄 정의 후보

- *"막연했던 노후 준비를, 분명한 숫자와 한 걸음으로 풀어드립니다."* (잠정)

### 7.4 랜딩 카피 분업 (잠정)

```
H1 (메인 슬로건):  막연한 노후, 분명한 한 걸음.
H2 (본질형):        부동산도 개별주도 아닙니다. 자산배분이면 충분해요.
든든 비교 섹션:    맡기지 않아도, 충분합니다.
기능 카피:          매달 5분, 직접 굴리는 노후 준비.
정상화 카피:        퇴직연금 90%는 디폴트 옵션에 그대로 둬요. 당신만 그런 게 아니에요.
```

### 7.5 안 쓰는 카피 결

- ❌ 불안 자극형 (*"X억 부족!"* *"이대로면 노후 파산"*)
- ❌ 입문자 취급형 (*"기초부터 알려드릴게요"*)
- ❌ 과장 수익 약속 (*"연 N% 보장"* *"두 배 만들기"*)
- ❌ 공격적 캠페인 (*"투자 안 하면 망한다"*)
- ❌ 자본시장법 경계 위반 결 (*"지금 매수"* *"이 종목"*)

## 8. 포트폴리오 큐레이션

### 8.1 1차 라인업 (3~4개 정적 자산배분)

- **K-올웨더형** (김성일 「마법의 돈굴리기」 참조)
- **글로벌 60/40**
- **30대 성장형** (주식 비중 70~80% 균형형)
- **영구포트폴리오** (해리 브라운 — 가장 보수적·균등)

### 8.2 명명·인용 원칙 — *법적 안전지대 유지*

- *일반명만 사용*. 인격 결합 X (퍼블리시티권 회피).
- 책 출처는 본문 *교양 콘텐츠*로 인용 (저작권법 인용 예외).
- 비중까지만 프리셋, **종목은 사용자가 카테고리별 후보 중 선택** (§v7 4.3 경계 준수).

### 8.3 추천 메커니즘

위험 성향 진단 2~3문항 → 1~2개 추천. 4개 비교 모드는 *고급 옵션*으로 분리.

### 8.4 동적 자산배분 (강환국 LAA·GTAA 등) — 1차 라인업 제외

매월 모멘텀 시그널 추적 필요 → *"덜 보게 만드는 앱"* 철학과 텐션. Phase 3 이후 검토.

## 9. 회사 퇴직연금 (DC) 다루는 방식

### 9.1 진입점으로의 가치

페르소나 B의 *방치 자산 1순위*. 든든·토스·뱅샐 누구도 안 건드리는 영역 (DC는 회사 운용 플랫폼과 묶여 일임 BM이 안 들어감).

### 9.2 다루는 방식

- **다루되 전면에 두진 않음.**
- 도토리의 *매매 지시서 패턴과 정합* — 실제 매매는 회사 운용 플랫폼에서, 운용 가이드는 도토리.
- 별도 SEO 페이지 *"DC 방치 진단"* 운영 → 회원가입 깔때기.

### 9.3 데이터 모델 결정 사항 (TBD)
