# 가을 숲 톤 가이드 — 디자인 시스템 적용

> **위상**: v8 §3.6 비주얼 톤 *"따뜻한 베이지·어스 톤 + 약간의 가을 숲 그린"*의 적용 가이드. 기존 디자인 시스템(`docs/design-system.md`, `src/app/globals.css`)을 *재해석*하여 v8 *감성/운영 화면 분리*(§3.4) 원칙에 매핑.
>
> **결론 미리**: **새 토큰 거의 필요 없음**. 기존 시스템이 이미 가을 숲 톤으로 잘 잡혀 있음. 핵심은 *적용 가이드*.

---

## 1. 기존 토큰 점검 — 이미 가을 숲에 와 있다

| 의도 | 현재 토큰 | 현재 컬러 | v8 §3.6 정합성 |
|---|---|---|---|
| 가을 숲 그린 (메인) | `--color-primary-*` (9단계) | `#4f7942` 등 forest green | ✅ 완벽 |
| 따뜻한 베이지·어스 (도토리 톤) | `--color-accent-*` (9단계) | `#c08b57` 등 warm amber | ✅ 완벽 (도토리/다람쥐 모피 톤) |
| 가을 낙엽 (강조·경고) | `--color-danger`, `--color-warning` | `#b24a3b` (단풍), `#c08b57` (밤) | ✅ 자연스러운 가을 결 |
| 가을 하늘 (정보) | `--color-info` | `#5b7a9a` | ✅ 차분한 결 |
| 따뜻한 중성 (배경·텍스트) | `--bg-base`, `--text-*` | `#faf9f6`, `#2d2a26` | ✅ 따뜻한 오프화이트 |

→ **결론**: v8이 요구하는 *베이지·어스·그린*은 이미 토큰 레이어에 충실하게 들어가 있음. 새 hex를 추가할 필요 없음.

---

## 2. 감성 vs 운영 화면 토큰 적용

v8 §3.4의 *감성/운영 분리*를 토큰 레벨에서 어떻게 구현할지.

### 2.1 감성 화면 (랜딩·온보딩·계산기 결과·분기 인사·축하)

**팔레트**:
- 메인 강조: `--color-primary-400` ~ `--color-primary-600` (밝은 forest green)
- 보조 강조: `--color-accent-300` ~ `--color-accent-500` (warm amber, 도토리 색)
- 배경: `--bg-base` 또는 `--color-accent-50` / `--color-primary-50` (살짝 따뜻한 톤)
- 텍스트 강조: `--color-foreground-strong`

**원칙**:
- *부드럽고 따뜻한 결*
- 둥근 모서리(`rounded-2xl`+) 적극
- 마스코트 일러스트 배치 OK
- 폰트 두께는 *가벼움* 우선 (`font-medium` 이하)

### 2.2 운영 화면 (매매 지시서·세금 한도·수익률·결제)

**팔레트**:
- 배경: `--bg-surface` 또는 `--bg-muted` (담백)
- 메인 강조: `--color-primary-700`+ (진한 forest)
- 데이터 표시: `--color-foreground` 위주, accent는 *수치 강조에만*
- 경고·위험: `--color-danger`, `--color-warning`

**원칙**:
- *정통·차분·읽기 쉬움*
- 모서리 살짝 둥글게 (`rounded-lg`)
- 마스코트 X
- 폰트 두께 `font-semibold` ~ `font-bold` (수치)
- 숫자에 `nums` (tabular)

### 2.3 한 화면에 둘이 섞일 때

대시보드처럼 한 화면 안에 *감성 카드*(축하, 분기 리포트 인사)와 *운영 카드*(매매 지시서, 세금 한도)가 공존하는 경우:

- 감성 카드: `bg-accent-50` / `bg-primary-50` 등 살짝 따뜻한 면
- 운영 카드: `bg-surface` (흰색에 가까운 담백)
- 같은 카드 안에서 *섞지 않기* — 카드 단위로 결을 분리

## 3. 마스코트 일러스트 색상 가이드

마스코트(다람쥐·도토리) 일러스트는 *감성 화면 한정*이며, 디자인 시스템 토큰을 *기반으로* 일러스트 컬러를 정의:

| 부위 | 권장 토큰 매핑 | 비고 |
|---|---|---|
| 다람쥐 모피 (메인) | `--color-accent-500` ~ `--color-accent-700` | warm amber 톤 |
| 다람쥐 배 (밝은 부분) | `--color-accent-100` ~ `--color-accent-200` | 크림 톤 |
| 다람쥐 눈·코 | `--text-strong` | 검정-갈색 |
| 도토리 모자 (cup) | `--color-primary-700` ~ `--color-primary-800` | forest green |
| 도토리 알맹이 | `--color-accent-500` ~ `--color-accent-600` | warm amber |
| 잎사귀 (배경 액센트) | `--color-primary-300` ~ `--color-primary-500` | sage |
| 떨어진 낙엽 (배경 액센트) | `--color-danger`, `--color-accent-700` | 단풍 |

> **운영 원칙**: 일러스트는 SVG로 작성하고 토큰을 직접 참조 (`fill="var(--color-accent-500)"`). hex 리터럴은 `acorn-icon.tsx` 등 *exempt 경로*에 한정.

## 4. 가능성 있는 신규 토큰 (TBD — 디자이너 검토)

기존 토큰으로 99% 커버되지만, *명시적인 의미 토큰*을 추가하면 카피·디자인 일관성이 더 좋아질 수 있는 후보:

| 신규 토큰 후보 | 의도 | 매핑 (제안) | 우선순위 |
|---|---|---|---|
| `--color-emotion-bg` | 감성 화면 카드 배경 | `var(--color-accent-50)` | 중 |
| `--color-operation-bg` | 운영 화면 카드 배경 | `var(--bg-surface)` | 중 |
| `--color-mascot-fur` | 마스코트 메인 모피 | `var(--color-accent-500)` | 낮 (일러스트에서 직접 참조 가능) |
| `--color-leaf-fall` | 가을 낙엽 강조 (분기 리포트 등) | `#c54a3b` (현재 danger) 변형 | 낮 |

> **권장**: 일단 *추가하지 않음*. 실제 화면 구현 후 *반복되는 패턴*이 보이면 그때 의미 토큰으로 추출.

## 5. 안티패턴 — 하지 말 것

- ❌ **중성 회색 배경**(`#f5f5f5` 등): 따뜻한 톤이 깨짐. `--bg-muted` (`#f4f2ec`)를 써야 가을 숲 결.
- ❌ **차가운 파랑 강조** (트위터 블루 등): 도토리 결 아님. `--color-info`(`#5b7a9a`)는 *차분한 가을 하늘*이라 OK.
- ❌ **네온/형광 그린**: forest green이 아님. `--color-primary-*` 9단계를 *벗어나지 않기*.
- ❌ **감성 화면에 흑백 대비**: 따뜻한 결 깨짐. *감성에서는 색을 입히기*.
- ❌ **운영 화면에 따뜻한 색 가득**: 신뢰·정확성 결 깨짐. *운영에서는 색 절제*.
- ❌ **마스코트를 아이콘 자리에**: 마스코트는 *감성 화면 1~2곳* 한정. 액션 버튼 옆 X.

## 6. 다크모드 동작

기존 `html.dark` 토큰이 *따뜻한 다크 베이지*(`#100f0c` 등)로 잡혀 있어 가을 숲 결이 다크모드에서도 유지됨.

- `--color-primary-*`, `--color-accent-*`: 라이트/다크 동일 (단계가 충분해서 다크에서도 잘 작동)
- 다크모드 추가 검토 필요 시: `html.dark` 안에서 *primary는 한 단계 밝게*, *accent는 두 단계 밝게* 매핑하는 변형 가능 (Phase 3 이후)

## 7. v8 카피 결과의 시각적 정합

| v8 카피 결 | 시각 표현 |
|---|---|
| 정상화·격려 (*"당신만 그런 게 아니에요"*) | warm amber 카드 배경, 둥근 모서리, 마스코트 작게 |
| 작은 승리 (*"한 발 시작했어요"*) | forest green 진행 바, 도토리 알맹이 아이콘 |
| 조용한 자신감 | 흰 배경 + 절제된 강조. 인용구 스타일 |
| 친구 톤 | 폰트 두께 가벼움, 줄간격 넉넉 (`leading-relaxed`+) |

---

## 8. 구현 시 ESLint·DS 정책 준수

- 차트·일러스트 외에는 hex 리터럴 금지 (CLAUDE.md DS 정책)
- 모든 색은 토큰 (`bg-primary-50`, `text-foreground-muted` 등) 사용
- 일러스트 SVG에서도 토큰 참조 권장 (`fill="var(--color-accent-500)"`) — 하드코딩보다 다크모드·테마 변경 대응 쉬움
