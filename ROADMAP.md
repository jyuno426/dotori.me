## 로드맵 개요

도토리의 개발은 크게 4단계로 나뉜다:

| 단계 | 기간 | 목표 |
|------|------|------|
| **Phase 1: MVP** | M1–M6 | 다계좌 통합 트래커 + 리밸런싱 + **세금 최적화 1차** + 배당 자동 입력 + 스크린샷 OCR → 클로즈드 베타 |
| **Phase 2: 성장** | M7–M12 | 유료 플랜 출시 + 세금 라우팅 고도화 + 은퇴 플래너 + 퍼블릭 출시 |
| **Phase 3: 확장** | Y2 | 고급 분석 + 세후 수익률 + 커뮤니티 |
| **Phase 4: 스케일** | Y3 | API 연동 + B2B + 글로벌 확장 검토 |

### M3–M4: 리밸런싱 계산기 + 월 적립 + **세금 최적화 Phase 1**

포트폴리오 트래커 위에 리밸런싱 엔진과 월 적립 배분 기능, 그리고 **한국 절세계좌 특화 핵심 차별점**인 세금 최적화 1차 기능을 구축한다.

#### 목표 비중 & 드리프트 감지

- [x] 종목별 목표 비중 설정 UI (M2에서 구현 완료)
- [ ] 드리프트 자동 감지 로직 (±3%, 5%, 10% 임계값)
- [ ] 드리프트 알림 (인앱 알림, 이메일/푸시 검토)

#### 리밸런싱 계산 엔진

- [x] ETF 1주 단위 매수·매도 수량 자동 계산
- [x] 매매 지시서 생성 (종목명, 수량, 현재가 정리)
- [ ] "매매 완료" 버튼 → 보유량 일괄 업데이트
- [ ] 예수금 보정 입력 필드

#### 월 적립 캐시플로우 리밸런싱

- [x] 부족 우선 모드 (특정 자산 5%+ 부족 시 집중 배분)

#### 세금 최적화 Phase 1 (이전: M7–M8에서 승격)

한국 절세계좌 특화가 도토리의 가장 큰 해자. 유료 전환 전에 베타에서 반드시 검증되어야 하는 축.

- [ ] **ISA 비과세 200만원 한도** 실시간 잔여 추적 (계좌별 YTD 합산)
- [ ] **IRP/연금저축 세액공제 한도** 연간 추적 (연금계좌 통합 900만, IRP 단독 300만 이내)
- [ ] **세액공제 한도 기반 월 적립 계좌 배분**: IRP → 연금저축 → ISA → 일반 우선순위
- [ ] **리밸런싱 계좌 라우팅 1차**: 같은 종목을 여러 계좌에 보유할 때 **매매는 ISA/연금계좌 우선**, 매도는 일반위탁 우선
- [ ] 라우팅 근거 설명 UI ("이 계좌에서 매매하는 이유")

#### 수익률 엔진 고도화

- [ ] 시간가중수익률 (TWR) 계산 엔진
- [ ] 금액가중수익률 (MWR) 계산 엔진
- [ ] 기간별 조회: 1개월, 3개월, YTD, 1년, 3년, 전체

#### 매매 지시서 플로우

- [ ] 매매 지시서 화면 (종목별 매수·매도 수량 리스트)
- [ ] MTS 전환 참조용 요약 뷰
- [ ] "완료" 후 보유수량 + 예수금 일괄 반영
- [ ] 리밸런싱 이력 기록

**M4 마일스톤:** 월 적립금 투입 시 세액공제 한도 우선 계좌 배분 매매 지시서를 받고, MTS에서 매매 후 5분 내 반영. ISA 비과세 한도 잔여가 대시보드에 상시 노출.

### M5: 배당 자동 입력 + **스크린샷 OCR** + 드리프트 알림

사용자의 수기 입력 부담을 줄이는 편의 기능. 모바일 퍼스트 차별화 경험의 핵심 구간.

#### 공공 API 연동 (배당 자동 감지)

- [ ] 공공데이터포털 주식배당정보 API 연동
- [ ] SEIBro Open Platform ETF 분배금 API 연동
- [ ] DART OpenAPI 배당 공시 연동
- [ ] 보유 종목 대상 배당/분배금 자동 감지 로직
- [ ] 감지 시 예수금 증가 레코드 자동 생성
- [ ] 자동 생성 레코드 사용자 확인 UI
- [ ] 불일치 시 예수금 오버라이드 흐름
- [ ] 감지 실패 시 폴백 (수기 입력 안내)

#### 스크린샷 OCR (모바일 차별화 경험)

증권사 앱 "보유 종목" 화면 스크린샷 1장으로 수량·예수금 일괄 업데이트. 스노우볼72·더리치 어디에도 없는 경험.

- [ ] 이미지 업로드 UI (모바일 PWA 최적화)
- [ ] Vision API 연동 (Claude Vision 또는 GCV 중 비용·정확도 검증 후 선택)
- [ ] 증권사별 파싱 프리셋 (키움·미래에셋·삼성·NH·한투·토스 — "종목명 + 수량 + 평가금액" 공통 구조)
- [ ] 파싱 결과 → 기존 수량/예수금 diff 확인 화면
- [ ] **프라이버시: OCR 후 이미지 즉시 폐기, 서버 저장 금지**
- [ ] **비용 통제: Free 월 1회 / Pro 월 30회 상한**

#### 드리프트 알림 ("덜 보게 만드는 앱" 철학 시작)

- [ ] ±5%p 편차 초과 시 주간 요약 이메일 발송 (Cloudflare Queue + Resend)
- [ ] 편차 임계값 이내면 *"이번 달 리밸런싱 필요 없음 ✓"* 홈 카드
- [ ] 월급일 입력 기반 적립 리마인더 (사용자 설정)

#### 데이터 이중화

- [ ] 복수 데이터 소스 간 크로스 체크
- [ ] API 장애 시 폴백 소스 자동 전환
- [ ] 데이터 수집 상태 모니터링 대시보드 (내부용)

**M5 마일스톤:** 배당/분배금이 자동 감지되고, 스크린샷 1장으로 보유 수량이 업데이트되며, 편차가 없으면 앱이 조용히 침묵한다.

#### 클로즈드 베타 준비

- [ ] 온보딩 플로우 구현 — **v8 5단계 사용자 여정 기준**
  - **메인 트랙: 투자 입문자 (페르소나 B)** — wake-up(은퇴 계산기 결과) → 납득(짧은 인라인 학습) → 선택(위험 성향 진단 → 1~2개 프리셋 추천) → 시작(보유 계좌 입력 + *없는 계좌는 인라인 가이드*) → 첫 매매 지시서

## 핵심 원칙

1. **최소 완전 데이터 모델:** 입출금 + 보유수량 + 예수금 잔액 3개만 기록. 수익률은 역산.
2. **증권사 API 미사용:** 법적 리스크 제로, 범용 구조 우선. 마이데이터 불가를 **프라이버시 강점으로 재프레임**.
3. **매매 지시서 패턴:** 계산·기록은 시스템, 매매 실행만 사용자.
4. **저비용 인프라:** Cloudflare 풀스택, 유휴 시 비용 0 수렴.
5. **세금 로직 모듈화:** 매년 세법 개정에 빠르게 대응. 한국 절세계좌 특화가 **도토리의 해자**.
6. **프레임워크가 곧 제품:** 입문자도 사용하면서 자연스럽게 자산배분 투자를 체득.
7. **"덜 보게 만드는 앱":** 월 2~4회 사용 목표. 편차가 없으면 침묵, 초과 시에만 알림. 증권사 앱과 정반대 축.
8. **유사투자자문업 경계 준수:** "비중 초과 알림"은 안전, "매수 시그널·종목 추천"은 금지. 자본시장법 제7조 3항 범위 내 계산·시각화 도구.
