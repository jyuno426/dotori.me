# 도토리 (DOTORI) — 개발 로드맵

> **자산배분 ETF 투자자를 위한 포트폴리오 트래커 + 리밸런싱 계산기 + 은퇴 플래너**
>
> dotori.me | "한 알씩 모아, 단단한 내일로"

---

## 로드맵 개요

도토리의 개발은 크게 4단계로 나뉜다:

| 단계 | 기간 | 목표 |
|------|------|------|
| **Phase 1: MVP** | M1–M6 | 포트폴리오 트래커 + 리밸런싱 계산기 + 배당 자동 입력 → 클로즈드 베타 |
| **Phase 2: 성장** | M7–M12 | 은퇴 플래너 + 리밸런싱 라우팅 + 유료 플랜 출시 |
| **Phase 3: 확장** | Y2 | 고급 분석 + 세후 수익률 + 커뮤니티 |
| **Phase 4: 스케일** | Y3 | API 연동 + B2B + 글로벌 확장 검토 |

### M3–M4: 리밸런싱 계산기 + 월 적립

포트폴리오 트래커 위에 리밸런싱 엔진과 월 적립 배분 기능을 구축한다.

#### 목표 비중 & 드리프트 감지

- [ ] 자산군별 목표 비중 설정 UI
- [ ] 드리프트 자동 감지 로직 (±3%, 5%, 10% 임계값)
- [ ] 드리프트 알림 (인앱 알림, 이메일/푸시 검토)

#### 리밸런싱 계산 엔진

- [ ] ETF 1주 단위 매수·매도 수량 자동 계산
- [ ] 매매 지시서 생성 (종목명, 수량, 현재가 정리)
- [ ] "매매 완료" 버튼 → 보유량 일괄 업데이트
- [ ] 예수금 보정 입력 필드

#### 월 적립 캐시플로우 리밸런싱

- [ ] 적립금 입력 → 부족 비중 자산 우선 배분 계산
- [ ] 균등 배분 모드 (드리프트 ±3% 이내)
- [ ] 부족 우선 모드 (특정 자산 5%+ 부족 시 집중 배분)
- [ ] 강제 리밸런싱 안내 (드리프트 10%+ 초과 시 매도 포함)
- [ ] 세액공제 한도 기반 계좌 배분: IRP(700만) → 연금저축(600만) → ISA → 일반

#### 매매 지시서 플로우

- [ ] 매매 지시서 화면 (종목별 매수·매도 수량 리스트)
- [ ] MTS 전환 참조용 요약 뷰
- [ ] "완료" 후 보유수량 + 예수금 일괄 반영
- [ ] 리밸런싱 이력 기록

**M4 마일스톤:** 월 적립금 투입 시 매매 지시서를 받고, MTS에서 매매 후 5분 내에 반영 완료.

## 핵심 원칙

1. **최소 완전 데이터 모델:** 입출금 + 보유수량 + 예수금 잔액 3개만 기록. 수익률은 역산.
2. **증권사 API 미사용:** 법적 리스크 제로, 범용 구조 우선.
3. **매매 지시서 패턴:** 계산·기록은 시스템, 매매 실행만 사용자.
4. **저비용 인프라:** Cloudflare 풀스택, 유휴 시 비용 0 수렴.
5. **세금 로직 모듈화:** 매년 세법 개정에 빠르게 대응.
6. **프레임워크가 곧 제품:** 입문자도 사용하면서 자연스럽게 자산배분 투자를 체득.
